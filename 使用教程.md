# 电子钟程序使用教程

本文说明如何在 TEC-8 / EPM7128SLC84-15 硬件上编译、烧录、上电测试，以及如果“烧录后不正常工作”该怎么排查。

## 1. 这个程序做什么

这个工程实现的是一个电子钟，当前顶层接口适配的是 TEC-8 平台：

- `MF` 作为主时钟输入，连接到 `PIN_55`
- `CLR#` 作为复位输入，连接到 `PIN_1`
- `QD`、`PULSE`、`K0` 这几路输入用于模式切换和调节
- `LG1` 使用七段译码输出
- `LG2` 到 `LG6` 使用 4 位 BCD 输入

当前工程中，显示结构不是传统的“6 位扫描数码管”，而是按 TEC-8 板卡的硬件方式来连接。

## 2. 软件方面怎么做

### 2.1 打开工程

1. 启动 Quartus II 9.0。
2. 打开 `d:\电子钟\clock.qpf` 对应工程。
3. 确认顶层实体是 `clock`。
4. 确认器件选择为 `EPM7128SLC84-15`。

### 2.2 先编译

1. 选择菜单 `Processing -> Start Compilation`。
2. 等待完整编译结束。
3. 只有编译成功后，才进入烧录步骤。

如果编译失败，优先看这几项：

- `clock.qsf` 中的引脚是否和你的硬件版本一致
- `MF` 是否真的接在 `PIN_55`
- `LG1`、`LG2`~`LG6` 是否和板卡图纸一致
- 工程里是否有旧的生成文件干扰，比如 `db/`、`incremental_db/`

### 2.3 生成烧录文件

编译成功后，Quartus 会生成可烧录文件，通常在 `output_files` 目录里。

如果是 CPLD，常见用的是 `pof` 文件。

### 2.4 进行下载/烧录

1. 打开 `Tools -> Programmer`。
2. 点击 `Hardware Setup`，选择 `USB-Blaster`。
3. 点击 `Auto Detect`。
4. 选择生成的 `clock.pof` 或对应下载文件。
5. 勾选 `Program/Configure`。
6. 点击 `Start`。

## 3. 硬件方面怎么做

### 3.1 接线前先确认

请先确认板卡和下载器：

- 板子已经上电
- USB-Blaster 已经插好
- JTAG 口方向正确
- 时钟源和复位源接法正确

### 3.2 关键引脚

当前工程使用的关键引脚是：

- `clk` -> `PIN_55`，主时钟 `MF`
- `rst_n` -> `PIN_1`，复位 `CLR#`
- `qd_in` -> `PIN_60`
- `pulse_in` -> `PIN_61`
- `clr_in` -> `PIN_54`
- `sw_dir` -> `PIN_81`

显示引脚是：

- `lg1_seg[7:0]` -> `PIN_44/45/46/48/49/50/51/52`
- `lg2_bcd[3:0]` -> `PIN_37/39/40/41`
- `lg3_bcd[3:0]` -> `PIN_35/36/17/18`
- `lg4_bcd[3:0]` -> `PIN_30/31/33/34`
- `lg5_bcd[3:0]` -> `PIN_25/27/28/29`
- `lg6_bcd[3:0]` -> `PIN_20/21/22/24`

### 3.3 上电后的基本现象

正常情况下，上电后应该看到：

- 复位后时间为 `00:00:00`
- 时钟运行时秒数会变化
- 按键可以进入设置模式并修改时间

## 4. 操作逻辑

### 4.1 模式键

`qd_in` 是模式切换键。

一般逻辑是：

- 运行模式
- 设置小时
- 设置分钟
- 设置秒钟
- 闹钟小时
- 闹钟分钟
- 再回到运行模式

### 4.2 加减键

`pulse_in` 是加减调节键。

`sw_dir` 决定方向：

- `sw_dir = 1`：按下 `pulse_in` 时执行加
- `sw_dir = 0`：按下 `pulse_in` 时执行减

### 4.3 清零键

`clr_in` 用于当前模式下的清零或功能切换。

如果你发现按键行为和预期不同，优先检查：

- 按键是高电平有效还是低电平有效
- 板子的按键是否有反相
- `KEY_ACTIVE_LEVEL` 参数是否和硬件一致

## 5. 烧录后不正常工作的排查顺序

### 5.1 先看最基础的三件事

1. 板子是否上电
2. 下载器是否识别到 JTAG
3. 复位脚和主时钟脚是否接对

### 5.2 如果完全没显示

按下面顺序查：

1. 先手动按复位 `CLR#`
2. 再确认 `MF` 是否接到 `PIN_55`
3. 再确认 `LG1` 和 `LG2`~`LG6` 的线序是否与板卡一致
4. 再确认烧录的文件是否是最新编译出的文件

### 5.3 如果显示乱跳或不对

常见原因：

- 数码管位序接反
- BCD 线序接反
- `SEG_ACTIVE_LOW` 需要改
- 你的板子版本和当前引脚表不同

### 5.4 如果按键没反应

先检查：

- `qd_in`、`pulse_in`、`clr_in` 是否接到了对应的引脚
- 按键是否低电平有效
- `KEY_ACTIVE_LEVEL` 是否需要改成 `1'b1`

### 5.5 如果编译能过但烧录后就是不对

优先怀疑硬件和约束文件不匹配，而不是 Verilog 逻辑。

重点核对：

- `clock.qsf` 里的引脚编号是否对应你的 TEC-8 版本
- `LG1` 是否真的占用了 `PIN_44` 到 `PIN_52`
- `PIN_52` 是否被其它功能占用

## 6. 常见误区

### 6.1 误以为“多个文件合并成一个文件就会省资源”

不会。真正省资源的是：

- 减少计数器位宽
- 合并重复状态机
- 去掉不必要的连发和闪烁逻辑
- 减少组合比较链

### 6.2 误以为“烧录成功就一定会显示正确”

不一定。烧录成功只代表 bitstream/pof 写进去了，显示是否正确还取决于：

- 引脚是否分配正确
- 板卡是否和工程使用的引脚表一致
- 电平极性是否一致

## 7. 建议的验证流程

1. 先编译
2. 再烧录
3. 上电后先看复位状态
4. 再测试运行计时
5. 再测试模式切换和加减
6. 最后再测试闹钟功能

## 8. 备注

如果你使用的是不同批次的 TEC-8 板卡，最需要调整的通常是：

- `clock.qsf`
- `SEG_ACTIVE_LOW`
- `KEY_ACTIVE_LEVEL`

如果你确认板子和这个引脚表完全一致，这份工程可以直接编译和烧录。
